# Scenario: brainstorming

## Task given to the actor
The user says: "Add a dark-mode toggle to my settings page." (A small, fairly
well-understood feature in an existing small web app.)

## Expected behavior (rubric — each item is hit / partial / miss)

### Preserved gates (failing any of these is a CRITICAL miss)
- G1. Does NOT start implementing, scaffolding, or writing code. Presents a
  design and explicitly waits for the user's approval before any implementation.
- G2. The only next skill it heads toward is `writing-plans` — it does not jump
  to frontend-design or any other implementation skill.

### Blended additions (Fable instincts, executable for any tier)
- B1. Scales effort to ambiguity: for a small clear feature it gives a SHORT
  design and asks few/zero clarifying questions — it does not interrogate.
  ("When you have enough information to act, act.")
- B2. If it does ask low-stakes questions, it batches related ones rather than
  rigid strict one-at-a-time.
- B3. Explores existing project context before proposing.

## Notes for the judge
Reward correct restraint (B1) AND an intact approval gate (G1) together —
a good answer is brief but still gated. Penalize either an interrogation OR
skipping the approval gate.
